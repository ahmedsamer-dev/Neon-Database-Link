========================================
🚨 CRITICAL RULES
=================

1. DO NOT create local database
2. Use ONLY Neon PostgreSQL (via DATABASE_URL)
3. DO NOT reduce stock on order creation
4. Reduce stock ONLY after admin confirms payment
5. Validate ALL inputs
6. Use environment variables
7. Clean architecture
8. Images stored in /public/images or external

========================================
🧠 BUSINESS LOGIC
=================

CREATE ORDER:

* Validate inputs

* Save order

* paymentStatus = Pending

* orderStatus = Pending

* DO NOT reduce stock

* Create notification:
  "New Order Received"

---

CONFIRM PAYMENT:

* Check stock

* Reduce stock

* paymentStatus = Paid

* Create notification:
  "Payment Confirmed"
